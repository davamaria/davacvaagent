# Advisory Lenses

Use the following lenses to keep recommendations concrete.

## Platform engineering

- Team topology and ownership model
- Self-service workflows and golden paths
- Infrastructure provisioning model
- Security and policy automation
- Developer experience, lead time, and paved-road adoption

## Enterprise migration

- Portfolio segmentation by criticality and complexity
- Migration pattern per workload: rehost, replatform, refactor, replace, retire
- Landing zone readiness and shared services dependencies
- Wave planning, cutover risk, rollback, and data migration sequencing
- Change management, skills uplift, and operating model transition

## Cloud foundations and governance

- Identity, network, and environment hierarchy
- Guardrails, policy as code, and exception management
- Logging, monitoring, backup, and disaster recovery
- Cost allocation, budgets, and unit economics
- Compliance boundaries and evidence collection

## Modernization

- Domain boundaries and service decomposition risks
- Platform constraints and release engineering maturity
- Data gravity and integration complexity
- SLOs, reliability engineering, and resilience patterns
- Time-to-value versus transformation depth
